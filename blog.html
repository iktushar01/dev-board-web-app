Question-1: What are the different ways to select an element in the DOM?

Different Ways to Select an Element in the DOM

1. By ID (getElementById)

You can use document.getElementById("myId") to select an element by its unique ID.

2. By Class Name (getElementsByClassName)

You can select multiple elements by class name using document.getElementsByClassName("myClass"). It returns an HTMLCollection.

3. By Tag Name (getElementsByTagName)

You can select all elements with a specific tag name using document.getElementsByTagName("p"). It returns an HTMLCollection.

4. By Name Attribute (getElementsByName)

You can select elements with a specific name attribute using document.getElementsByName("myName"). It returns a NodeList.

5. By CSS Selectors (querySelector & querySelectorAll)

querySelector selects the first matching element: document.querySelector(".myClass")
querySelectorAll selects all matching elements: document.querySelectorAll("div.myClass"). These methods return a NodeList.

6. Using document.body or document.documentElement

You can also select the document.body or document.documentElement to select the <body> or <html> element, respectively.

Use querySelector and querySelectorAll for maximum flexibility.

Question-2: What is the difference between innerHTML, innerText, and textContent ?

Difference Between innerHTML, innerText, and textContent

1. innerHTML

The innerHTML property allows you to get or set the HTML content inside an element, including any HTML tags and elements. It will also render HTML content, like images, links, etc.
document.getElementById("myElement").innerHTML

2. innerText

The innerText property returns the text content of an element, but it excludes any HTML tags and will return the visible text on the screen. It respects the CSS styling, such as visibility:hidden or display:none.
document.getElementById("myElement").innerText

3. textContent

The textContent property also returns the text content of an element, but unlike innerText, it does not respect CSS styling. It returns the raw text content of the element, including text hidden by CSS.
document.getElementById("myElement").textContent

Key Differences

  • innerHTML gives HTML content and renders it (including tags).
  • innerText provides visible text and respects CSS styles.
  • textContent returns all the text, including hidden content, and ignores CSS styles.

In summary, use innerHTML for HTML content, innerText for visible text, and textContent for raw text regardless of CSS styling.

Question-3: What is event delegation in the DOM?

Event Delegation in the DOM

What is Event Delegation?

Event delegation is a technique in JavaScript where you attach a single event listener to a parent element to handle events for its child elements. Instead of attaching individual event listeners to each child element, you delegate the handling of the events to the parent.

Why Use Event Delegation?

Event delegation is efficient, especially when dealing with dynamically added elements or a large number of elements. By attaching an event listener to a parent, you reduce memory usage and simplify your code.

How Does It Work?

The idea is to take advantage of event bubbling. When an event is triggered on a child element, it bubbles up to the parent element. The parent can then determine which child element triggered the event and handle it accordingly.

Example of Event Delegation

                      
          document.querySelector('.parent').addEventListener('click', function(event) {
            if (event.target && event.target.matches('button.child')) {
              alert('Button clicked!');
            }
          });

In the above example, the event listener is attached to the parent element with the class .parent. It listens for click events on any button.child inside the parent.

Key Benefits of Event Delegation

  • Improves performance by reducing the number of event listeners.
  • Handles dynamically added elements without needing to reattach event listeners.
  • Simplifies the code by reducing repetitive event listener code.

Event delegation helps in managing events efficiently and makes your code more scalable and maintainable.

Question-4: What is event bubbling in the DOM?

Event Bubbling in the DOM

What is Event Bubbling?

Event bubbling is a concept in JavaScript where an event that occurs on a child element bubbles up through its ancestors in the DOM. The event propagates from the innermost element (the target) all the way up to the root element (usually the document object).

How Does Event Bubbling Work?

When an event (like a click) occurs on a child element, the event doesn't just stay on that element; it bubbles up to its parent, then to its parent’s parent, and so on. Each parent has the opportunity to handle the event, which makes it possible to attach a single event listener on a parent element to handle events for multiple child elements.

Example of Event Bubbling

                      
          document.querySelector('.parent').addEventListener('click', function() {
            alert('Parent element clicked!');
          });
          
          document.querySelector('.child').addEventListener('click', function(event) {
            alert('Child element clicked!');
            event.stopPropagation(); // Stops the event from bubbling up
          });

In the above example, if the child element is clicked, the child’s click event will be triggered first. The event then bubbles up to the parent element, triggering the parent’s click event. We can use event.stopPropagation() to stop the event from bubbling further up.

Key Characteristics of Event Bubbling

  • Event starts at the target element and bubbles up to the root of the DOM.
  • Multiple event listeners can be attached to different levels in the DOM hierarchy.
  • It is the default behavior for most events in the DOM (like click, keypress, etc.).
  • Using event.stopPropagation(), you can prevent the event from bubbling up.

Event bubbling is an essential concept for efficient event handling in the DOM, enabling you to manage events at the parent level rather than on individual child elements.

Question-5: How do you create, add, and remove elements using JavaScript?

Create, Add, and Remove Elements using JavaScript

Creating an Element in JavaScript

To create a new element in JavaScript, you can use the document.createElement() method. This method allows you to create an HTML element that you can later add to the DOM. 

                      
          const newDiv = document.createElement('div'); // Creates a new div element
          newDiv.textContent = 'This is a new div!'; // Adds text to the div
          document.body.appendChild(newDiv); // Adds the new div to the body of the document

Adding an Element to the DOM

Once you create an element, you can add it to the DOM using methods like appendChild() or insertBefore(). The appendChild() method adds a new child element to the end of the specified parent element. 

                      
          const parentDiv = document.querySelector('#parent'); // Selects a parent element
          parentDiv.appendChild(newDiv); // Adds the new div to the parent element

Removing an Element from the DOM

To remove an element from the DOM, you can use the removeChild() method. This method removes a specified child element from a parent element. 

                      
          parentDiv.removeChild(newDiv); // Removes the new div from the parent element

Example: Create, Add, and Remove an Element

Below is a complete example demonstrating how to create, add, and remove an element using JavaScript. 

                      
          const newDiv = document.createElement('div');
          newDiv.textContent = 'Click me to remove!';
          newDiv.style.backgroundColor = '#ffcc00';
          newDiv.style.padding = '20px';
          newDiv.style.margin = '10px';
          document.body.appendChild(newDiv);
          
          // Adding an event listener to remove the element when clicked
          newDiv.addEventListener('click', function() {
            document.body.removeChild(newDiv); // Removes the div when clicked
          });

Summary

To create, add, and remove elements in the DOM, use document.createElement() to create an element, appendChild() or insertBefore() to add it to the DOM, and removeChild() to remove it from the DOM.

JavaScript DOM manipulation methods like createElement(), appendChild(), and removeChild() are fundamental for dynamically altering web page content.
